# DeWeb常见问题 FAQ

### Q：DeWeb系统的并发怎么样？
**A**：单服务器大概200（当时测试225）并发，如果采用nginx做反向代理，可以达到1000+


### Q：DeWeb和WebXone的区别？
**A**：

**相同点：**
- 两者为同一开发者研发.QQ:45300355,碧树西风
- 都是为了解决Delphi开发Web的问题

**不同点：**
- WebXone采用的ActiveX/NPAPI技术，要求客户端必须为Windows,所以不支持手机/平板
DeWeb采用的VUE技术，支持所有平台，包括电脑/手机/平板等
- WebXone采用的ActiveX/NPAPI技术在安全上有一定隐患，目前已被大多数浏览器禁用，所以WebXone仅支持较老的浏览器
DeWeb支持所有现代浏览器，推荐Chrome/FireFox/Edge,越新越好
- WebXone几乎支持所有控件，包括第三方；
DeWeb目前支持部分基础控件（基本够用），也提供了拓展机制，但总的支持控件是有限的
- WebXone目前已停止研发
DeWeb目前正在大力发展中，还可以拓展到APP、小程序、公众号等

### Q：DeWeb和unigui的区别？
**A**：

**相同点：**
- 都是采用Delphi开发网页的平台

**不同点：**
- DeWeb不需要安装控件， 而unigui需要安装自己的控件
- DeWeb无需要学习HTML/CSS/JavaScript, 而Unigui需要学习
- DeWeb为国产自主研发；Unigui为国外产品

**DeWeb的优点：**
- 界面美观大方！ 采用目前最流行的UI框架之一：ElemnentUI
- 快速有效的技术支持！可通过QQ/email/电话交流！ 需要时也可以远程 
- 持续快速研发！
