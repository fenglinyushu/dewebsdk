**基本规则**
___
- 基本和原生Delphi一样开发  
DeWeb开发其实就是采用Delphi开发一个特殊的DLL，所以开发过程基本和原生Delphi一样。  
不过需要注意以下内容：
- 可视控件只使用DeWeb支持的控件，
- 不可视控件都可以使用
- 不能使用任何模式窗体或函数，  
如ShowModal, ShowMessage, MessageDlg, InputQuery等。  
此类操作需要使用DeWeb对应的替代函数；
- 不能使用全局变量，如需要使用，可以通过Form1的Public变量进行传递
- 主窗体Name必须为“Form1”
- 其他窗体Name必须以“Form”起始，且声明必须放置到Form1的public中，如：
```pascal
type
  TForm1 = class(TForm)
    PageControl1: TPageControl;
    TabSheet1: TTabSheet;
    TabSheet2: TTabSheet;
    TabSheet3: TTabSheet;
  public
    Form2: TForm2;
    Form3: TForm3;
    Form4: TForm4;

    gsMainDir   : String;
    goConnection    : TADOConnection;
  end;
```  
- 每个事件只能结束时在前端返回一个结果。也就是在一个事件中多次更新一个值，在前端只会看到最后的值。
```pascal
begin
	Label1.Caption	:= 'Loading...';
	//
	Button1.OnClick(Self);
	//
	Label1.Caption	:= 'Success';
end;
```  
以上过程中，前端不会先出现loading..., 再出现Success,而是只显示Success
- 除dpr单元外的unit中不能使用Form1来取得对象。 
在Form1所在窗体单元（一般为unit1.pas）中, 可以使用self;  
在工程内其他窗体中，需要使用TForm1(Self.Owner)来取得Form1对象
- 工程内其他窗体必须以Form1为Owner创建

- end.
